Detailed Description

template<typename DiGraph, typename Domain>
class NegCycleFinderQ< DiGraph, Domain >

Negative Cycle Finder with constraints using Howard's method.

This class extends the basic negative cycle detection to support constrained optimization problems. It implements both predecessor and successor versions of Howard's algorithm, providing flexibility in how cycles are detected and how distance updates are constrained, via an update_ok callback.

Algorithm variants:

  1. Predecessor-based (howard_pred):
    • Traditional Bellman-Ford relaxation
    • Updates dist[v] based on dist[u] + weight(u,v)
  2. Successor-based (howard_succ):
    • Reverse relaxation logic
    • Updates dist[u] based on dist[v] - weight(u,v)
